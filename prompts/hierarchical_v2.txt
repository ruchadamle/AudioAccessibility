You are generating a structured accessibility description of a bar chart for a blind or low-vision user.

Output raw JSON only.
Do not wrap the JSON in Markdown.
Do not use code fences.
Do not include any explanation before or after the JSON.

Return valid JSON matching this exact schema:

{
  "chart_type": "",
  "title": "",
  "axes": {
    "x_axis_title": "",
    "y_axis_title": "",
    "y_axis_unit": ""
  },
  "key_values": [],
  "comparisons": [],
  "trends": [],
  "takeaway": ""
}

Field requirements:
- "chart_type": use only one of these natural labels:
  - "simple bar chart"
  - "grouped bar chart"
  - "stacked bar chart"
- "title": copy the visible chart title if present, otherwise use an empty string.
- "axes.x_axis_title": use a human-readable x-axis label if it is visible or strongly supported by the chart. Do not blindly copy awkward text.
- "axes.y_axis_title": use a human-readable y-axis label if it is visible or strongly supported by the chart.
- "axes.y_axis_unit": use a human-readable unit only when it is safely inferable from the chart. Otherwise use an empty string.
- "key_values": include 2 to 3 short factual standalone sentences that are easy to read directly in prose.
- "comparisons": include 1 to 3 short factual standalone sentences that add useful relationships and do not simply repeat key_values.
- "trends": include 0 to 2 short factual standalone sentences only if a meaningful overall pattern is visible.
- "takeaway": include one short, natural sentence summarizing the main message without repeating the title.

Axis guidance:
- Prefer labels that help a human understand the chart rather than labels copied literally from the image.
- If the chart clearly shows percentages, prefer a human-readable label such as:
  - "Percentage"
  - "Percent"
  and use "percent" as the unit when appropriate.
- If the visible y-axis label is generic, such as "Data Value", prefer a clearer interpretation only when the chart strongly supports it.
- If the chart does not safely support a clearer interpretation, keep the label general rather than hallucinating.
- Do not invent units that are not supported by the chart.

Stacked quartile guidance:
- For stacked bar charts with titles like "County distribution by X quartiles across states", interpret the chart as showing how counties are distributed across quartiles of X for each state.
- These charts show total county counts and quartile composition, not direct counts of people with the condition.
- Describe total county counts separately from quartile composition.
- Do not write misleading statements such as:
  - "Mississippi has the highest count of arthritis among adults"
- Prefer statements such as:
  - "Georgia has the highest total county count at approximately 160"
  - "Mississippi and Oklahoma have larger shares in the highest quartile (Q4)"
  - "The chart shows how counties are distributed across quartiles for arthritis prevalence across states"

Writing guidance:
- Be factual, concise, and specific.
- Prefer approximate numeric values when exact values are not clearly labeled.
- Use short standalone sentences that can be rendered smoothly into a readable paragraph.
- Avoid redundant facts across key_values, comparisons, trends, and takeaway.
- Avoid repeating the chart title, axis labels, or the same numeric fact in multiple fields.
- Use natural phrasing that sounds good when read aloud.
- Do not hallucinate missing labels, values, units, or trends.
- Output must be valid JSON only.
