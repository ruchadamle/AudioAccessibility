You are generating a structured accessibility description of a bar chart for a blind or low-vision user.

Return ONLY valid JSON.
Do not include Markdown.
Do not include code fences.
Do not include any explanation before or after the JSON.

Use this exact schema:

{
  "chart_type": "",
  "title": "",
  "axes": {
    "x_axis_title": "",
    "y_axis_title": "",
    "y_axis_unit": ""
  },
  "key_values": [],
  "comparisons": [],
  "trends": [],
  "takeaway": ""
}

Field requirements:
- "chart_type": identify the chart type, such as "simple bar chart", "grouped bar chart", or "stacked bar chart".
- "title": the chart title if visible, otherwise use an empty string.
- "axes.x_axis_title": the x-axis title if visible, otherwise use an empty string.
- "axes.y_axis_title": the y-axis title if visible, otherwise use an empty string.
- "axes.y_axis_unit": the y-axis unit if visible, such as "percent", "percentage", "dollars", or another unit. If not visible, use an empty string.
- "key_values": a list of short factual strings with the most important values or categories. Include the largest category and the smallest category if they are clear. Include one additional notable value when possible.
- "comparisons": a list of short factual comparison statements, such as which category is larger or whether one value is about twice another. Only include comparisons that are clearly supported by the chart.
- "trends": a list of short factual trend statements only when a meaningful overall pattern is visible. If no clear trend exists, return an empty list.
- "takeaway": one short sentence summarizing the main message of the chart.

Rules:
- Use empty strings or empty arrays when information is unavailable.
- Be factual and concise.
- Do not hallucinate missing values, labels, or trends.
- Do not describe anything outside the chart.
- Output must be valid JSON only.
