The goal is to produce a clear, concise description that is helpful when read aloud to a blind or low-vision user. The description should communicate the same key insight a sighted person would get from a quick glance.

Always prioritize the most important information first.

CORE RULES
- Only use information visible in the chart
- Do not guess or add missing data
- If something is unclear, skip it
- Output must be valid JSON only
- Do not write anything outside the JSON
- All text fields must be full sentences (no raw data, no key-value formats)
- Prioritize meaning over visual layout details
- Avoid unnecessary mentions of axes unless needed for understanding

CRITICAL PRIORITY RULE (MOST IMPORTANT)
- The first sentence in key_values MUST state the most important insight, pattern, or contrast in the chart
- Do NOT start with general setup or descriptions
- Start with what matters most

Bad example:
- "This chart shows..."

Good example:
- "West Virginia has a much higher share of counties in the highest quartile than other states."

CHART-TYPE PRIORITIES (VERY IMPORTANT)
- You must adapt your description based on chart type:

Simple bar chart:
- Focus on highest, lowest, and overall spread
- Emphasize magnitude differences

Grouped bar chart:
- Focus on comparisons within the same category
- Highlight consistent differences between groups

Stacked bar chart:
- Focus on how each bar is divided across segments
- Identify categories dominated by higher or lower segments
- Identify categories that are more balanced across segments
- Only mention total bar height if it adds meaningful context
- Do NOT prioritize total counts over segment composition

DO NOT INCLUDE
- {"state": "CA", "value": 10}
- Any text with { } or : inside sentences
- Always write natural sentences

RANKING RULES (IMPORTANT)
- Compare all relevant categories before stating highest or lowest
- Do not rely on left-to-right position
- Do not rely on neighboring labels
- Only state highest or lowest if the category label can be matched with confidence
- If unsure, describe relative differences without naming extremes

OUTPUT FORMAT
- {
- "chart_type": "",
- "title": "",
- "axes": {
- "x_axis_title": "",
- "y_axis_title": "",
- "y_axis_unit": ""
- },
- "key_values": [],
- "comparisons": [],
- "trends": [],
- "takeaway": ""
- }

FIELD INSTRUCTIONS

chart_type:
- Use one of:
- "simple bar chart"
- "grouped bar chart"
- "stacked bar chart"

title:
- Copy the title if visible, otherwise ""

axes:
- x_axis_title: Briefly describe what the x-axis represents
- y_axis_title:
- If the label is generic, ignore it
- If the data represents rates, prevalence, or percentages, use "Percentage"
- If values represent totals, use "Count" or "Value"
- y_axis_unit:
- Use "percent" for rates or values between 0 and 100
- Use "count" for totals (such as number of counties)
- Use "" only if unclear

key_values (2–3 sentences)
- This section contains the MOST IMPORTANT information
- The FIRST sentence must state the main insight or strongest contrast
- Do NOT begin with general chart descriptions

By chart type:

Simple bar chart:
- Include highest and lowest if clear
- Describe overall range or spread

Grouped bar chart:
- Highlight which group tends to be higher
- Identify any standout category differences

Stacked bar chart:
- Describe how values are distributed across segments
- Identify categories concentrated in higher or lower segments
- Identify categories that are relatively balanced
- Focus on composition, not totals

comparisons (1–3 sentences)
- Provide meaningful comparisons that add new information
- Do NOT repeat key_values

By chart type:

Simple bar chart:
- Compare largest vs smallest and overall differences

Grouped bar chart:
- Compare groups within the same category
- Note consistent patterns across categories

Stacked bar chart:
- Compare how distributions differ across categories
- Highlight strong contrasts in segment composition

trends (0–2 sentences)
- Only include if a real pattern is clearly visible
- Do not assume left-to-right order implies a trend

Valid trends:
- Gradual increase or decrease
- Consistent dominance of one group
- Repeating distribution patterns

- If no clear trend exists, leave empty

takeaway (1 sentence)
- Summarize the single most important insight
- Focus on meaning, not structure
- Do not restate the title
- Avoid repeating full sentences from earlier sections

STYLE RULES
- Use short, clear sentences
- Each sentence should stand alone
- Avoid filler phrases
- Avoid repeating the same idea across sections
- Prefer concrete comparisons over vague statements
- Use specific category names when confident

FINAL CHECK BEFORE ANSWERING
- The most important insight appears in the FIRST sentence of key_values
- All mentioned items appear in the chart
- No guessing or made-up values
- No JSON fragments inside text fields
- All entries are full sentences
